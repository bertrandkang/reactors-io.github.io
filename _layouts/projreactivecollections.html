  • Event-driven

    Event-driven architecture targeted at building reactive applications with time-varying inputs. Ensures a better separation of concerns and supports layered applications.

  • Declarative

    Declarative programming style based on reactive composition for more concise and understandable programs. Relies on higher-order dataflow to change data dependencies dynamically.

  • Concurrent

    Concurrency by separating parallel event propagations into entities called isolates. Isolates communicate by exchanging events through typed channels.

  • Reactive containers

    Reactive data containers for reactive mutable data. Reactive container queries are much more efficient than their stream-based counterparts.

  • Resilient

    Avoids unnecessary synchronization and buffering to achieve high throughput. Comes with idiomatic support for failure-handling inside isolates.

  • GC-friendly

    Uses Scala Type-Driven Specialization to eliminate boxing for primitive values. Promotes reactive modifications on mutable data to avoid excess garbage. GC pauses never were shorter!
