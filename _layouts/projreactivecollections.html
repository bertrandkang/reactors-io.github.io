Event-driven

Event-driven architecture serves as the basis for distributed computing.

Events are encapsulated inside entities called event streams. After an event arrives, it gets forwarded to the observers of that event stream. 

// Create an event stream.
val events = new Events.Emitter[Int]

// Compute sum of events.
var sum = 0
events.onEvent { x => sum += x }

// Count the events.
var count = 0
events.on { count += 1 }

// When done, compute the average.
var avg = 0
events.onDone { avg = sum / count }

// Emit a few events and close the event stream.
for (i <- List(1, 2, 6)) events.react(i)
events.unreact()

First-class

First-class events streams and signals allow a better separation.

Components are expressed as event stream operations, which can be packaged into modules, and later reused and composed. 

// Define digital circuits.
def and(a: Signal[Boolean], b: Signal[Boolean]) =
  (a zip b) { _ && _ }
def xor(a: Signal[Boolean], b: Signal[Boolean]) =
  (a zip b) { _ ^ _ }
def halfAdder(a: Signal[Boolean], b: Signal[Boolean]) =
  (xor(a, b), and(a, b))
def probe(name: String, r: Events[Boolean]) =
  r.onEvent(v => println(s"$name: $v"))

// Simulate a half-adder.
val inputA = RCell(false)
val inputB = RCell(false)
val (sum, carry) = halfAdder(inputA, inputB)
probe("sum", sum)
probe("carry", carry)
inputA := true
inputB := true

Declarative

Declarative programming style based on functional value composition yields programs that are more concise and easier to understand. 

// Fetch contents of a URL
// or fail after a 10 seconds countdown.
val timer = system.time.periodic(1.second)
  .map(_ => 1)
  .scanPast(4)(_ - _)
  .takeWhile(_ >= 0)
system.net.resource.string("http://www.ietf.org/rfc/rfc1738.txt")
  .map(_.toString)
  .until(timer.unreacted)
  .ivar
  .orElse("Request failed.")
  .onEvent { txt =>
    println(txt.take(512) + "...")
    main.seal()
  }

Concurrent

Concurrency is achieved by separating event propagations into entities called isolates. Isolates communicate through typed channels. 
// Client asks for the server time
class TimeServer extends Iso[(String, Channel[Long])] {
  main.events onMatch {
    case ("time", response) => response ! System.currentTimeMillis()
  }
}
class Client(server: Channel[(String, Channel[Long])])
extends Iso[Long] {
  sysEvents.onMatch {
    case IsoStarted => server ! ("time", channel)
  }
  main.events onEvent {
    time => println("Server Unix time: " + time)
  }
}

val timeServer = system.isolate(Proto[TimeServer])
val client = system.isolate(Proto[Client](timeServer))

Reactive containers

Reactive data containers contain reactive mutable data. Events are triggered when parts of the dataset change. 
// Store ranks of a starship crew and establish the rules
val rank = RMap[String, String]
rank("Geordie") = "Lieutenant"
rank("Deanna") = "Lieutenant"
rank("Data") = "Commander Lieutenant"
rank("Riker") = "Commander"

val numLt: Events[Int] =
  rank.react.filter(_._2 == "Lieutenant").size
val shipRank = new Commutoid[(String, String)] {
  val zero = ("nobody", "no-rank")
  val operator = { (x, y) =>
    if (x._2 < y._2) x else y
  }
}
val commandingOfficer: Events[String] =
  rank.react.commuteFold(shipRank)

(commandingOfficer zip numLt)((m, n) => (m, n)).onEvent { (m, n) =>
  println(s"${m._1} says: all $n lieutenants shall mop floors!")
}

rank("Picard") = "Captain"
rank("Worf") = "Lieutenant"

GC-friendly

Eliminates boxing for primitive values such as integers and doubles. GC pauses were never shorter! 
// No boxing!
val cell = RCell[Int](0)
val evens = cell.filter(_ % 2 == 0).to[RSet[Int]]
val odds = cell.filter(_ % 2 == 1).to[RSet[Int]]
cell := 1
cell := 2
cell := 3
cell := 4
(evens union odds).foreach(println)
